# 焦点导航与深灰背景设计

## 目标

- 遥控器四向导航始终落在可见应用卡片上，不允许焦点进入 `GridView` 容器或空白单元。
- 向左到第一列、向右到最后一个应用、向上到第一行、向下没有应用时，消费按键并保持当前卡片激活。
- 默认背景使用极小的深灰朦胧位图，避免风景壁纸抢占视觉和运行时模糊开销。

## 根因

当前普通导航交给 Android `GridView` 的 `focusSearch`。小米 Android 9 在列表自身与可聚焦子卡片之间切换时，会保留选择索引却把真实焦点交回容器，形成“索引能移动、卡片无法激活”的不可见焦点状态。卡片尺寸大于焦点轨道时，该问题更明显。

## 焦点模型

`LauncherActivity` 在非移动模式下拦截四个方向键。目标索引只根据当前应用索引、列数和应用总数计算：

- 左右不跨行。
- 上下按列移动，目标必须对应真实应用。
- 无有效目标时返回当前索引。
- 每次按键后同时调用 `GridView.setSelection()` 和可见卡片 `requestFocus()`。
- `GridView` 自身不作为普通桌面的最终焦点；数据刷新完成后仍将焦点恢复到真实卡片。

索引计算放入纯 Java `GridFocusNavigator`，覆盖首项、末项、不完整末行和上下边界测试。

## 默认背景

新增约 `160x90` 的深灰色朦胧 WebP/PNG 资源，只有柔和亮度变化，不包含可辨识景物。默认路径直接加载该小图并由 `ImageView` 双线性放大，不执行解码后再次缩放。用户主动选择的自定义壁纸继续沿用现有低分辨率柔化路径。

## 性能与兼容性

- 不引入 RecyclerView 或第三方依赖。
- 每次按键仅做常数次整数计算和一次焦点请求。
- 保持 Android 6 / API 23 下限。
- 构建、单元测试、lint、R8 和签名仅在 GitHub Actions 执行。

## 验证

- 单元测试验证所有方向边界和不完整末行。
- 在 Xiaomi TV Box S 上连续执行左、右、上、下，确认始终存在且只有一个卡片焦点。
- 检查 HOME、菜单设置返回、长按入口及日志，无崩溃或 ANR。
- 使用 `gfxinfo` 检查方向键动画卡顿帧。
