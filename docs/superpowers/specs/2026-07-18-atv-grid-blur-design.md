# Apple TV 网格与柔化壁纸设计规格

## 目标

在 Android 9、1920x1080 的 Xiaomi TV 盒子上，把桌面调整到参考图的紧凑 Apple TV 网格：默认六列、四行完整可见、所有 Banner 都有一致大圆角，并用低成本柔化壁纸突出卡片。

## 网格几何

- 默认六列不变。
- 横向边距为屏宽约 4%，横向间距为屏宽约 2.5%。
- 1920 宽输出时，边距约 77px、间距约 48px、每个轨道约 254px。
- 卡片保持 16:9；默认卡片约 254x143px。
- 网格顶部 padding 为屏高约 6.7%；卡片在条目内部再保留 8dp 焦点阴影空间，因此首行卡片顶部约 88px。
- 条目只保留 8dp 顶部空间、22dp 名称高度、4dp 名称间距和 8dp 底部空间；纵向间距 10dp。1080p 下完整显示四行。
- `GridView` 显式设置列宽和 `STRETCH_SPACING`，每个可聚焦条目对应一个真实卡片轨道，不保留不可见焦点占位。
- 卡片尺寸设置只改变轨道内图片尺寸，轨道宽度和方向键位置保持稳定。

## 圆角

- 保留 `ImageView` outline 作为 elevation 阴影轮廓。
- Banner 在后台生成 Bitmap 时使用高度 12% 的圆角路径裁切，透明角直接写入缓存结果。
- 缓存键增加渲染版本；升级后不读取旧的方角磁盘缓存。

## 壁纸柔化

- 不使用逐帧模糊、RenderScript、RenderEffect 或第三方库。
- 默认壁纸和自定义壁纸都在现有单线程后台执行器中解码。
- 解码阶段限制长边到约 320px，再以双线性过滤缩小到长边约 160px；`ImageView` 由 GPU 放大至全屏，形成稳定柔化效果。
- 布局不再通过 `android:src` 同步解码完整默认壁纸。
- 保留暗色 scrim，增强图标对比度。

## 性能与错误处理

- 焦点移动不触发壁纸处理、Banner 解码或网格重算。
- 柔化壁纸常驻 Bitmap 约 160x90，ARGB_8888 内存约 56KB。
- Banner 仍由两个有界后台线程处理，圆角裁切只在缓存未命中时执行一次。
- 壁纸读取失败时后台解码内置壁纸；两者都失败时保持纯色背景，不崩溃。

## 验收

- 1920x1080 默认六列时完整看到四行，方向键只落在真实应用卡片上。
- 方角 APK Banner 在像素层面变为一致大圆角。
- 壁纸明显柔化且桌面静止时没有持续 CPU/GPU 工作。
- 菜单键设置往返、HOME 启动、应用启动和移动模式保持正常。
- GitHub Actions 的 release 单测、lint、R8、签名验证通过，并在已连接盒子上截图和日志复测。
